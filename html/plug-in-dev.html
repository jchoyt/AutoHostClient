Stars! AutoHost Client
Plug-in Developers Guide

Steve Leach
15 October 2004

Introduction

The Stars! Autohost client (AHC) can be extended in a number of ways using "plug-ins". A plug-in is a Java class (or classes) that implement specific interfaces and are packaged in a way that allows the client to find, load and initialize them.

Installing Plug-ins

To use a packaged plug-in, simply copy the JAR file into a "plugins" subdirectory of the directory that AHC is run from. When AHC is next started, all plugins in the JAR file will be loaded automatically.

Types of Plug-in

Game Panel Button

Game Panel Button plug-ins add a new button to the game information panel.

Map Layer

Map Layer plug-ins are used by the Map game panel button plug-in to draw the map details.

Analyzer Report

Report plug-ins are loaded by the Analyzer Report game panel button plug-in.

Writing a Plug-in

Developing a plug-in is mainly a matter of writing a class that implements one of the plug-in interfaces. Some interfaces also have abstract classes defined that implement much of the functionality, in which case it may be easier to subclass the abstract class.

Report Plugins

Report plug-ins implement the interface stars.ahc.plugins.analyzer.AnalyzerReport or extend the abstract class stars.ahc.plugins.analyzer.AbstractAnalyzerReport. The latter method is recommended as the abstract class provides a number of utility methods that make report development much easier.
